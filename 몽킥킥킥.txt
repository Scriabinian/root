@파일 입출력.

-자바 입출력 스트림(InputStream/OutputStream)
자바 프로그래밍이 디스크에 접근해서 파일을 읽고 쓰는 통로를 스트림이라고 한당.
1.스트림은 입출력 장치와 자바프로그램에 양끝에서 전달된당.
2.단방향 통신이며, 입력과 출력에 의해서 스트림은 독립적으로 각각 움직인당.
3.연속된 데이터의 흐름으로 입출력 진행 시 다른작업을 할 수 없는 상태가 된다.(입출력이 인터럽트의 최우선 단위가 된다);

@입출력 스트림의 종류.
입출력 패키지는  java.io 클래스에서 제공되며, 파일 시스템의 정보를 얻기 위해서 File크래스와 데이터 입출력을 위한 다양한 스트림 클래스를 제공한다

1. 바이트 기반 스트림.
파일 형식에 구애받지 않고 모든 파일을 읽고 쓸 수 있으며, 바이트 기반 입력 스트림의 최상위 클래스는 InputStream 이며, 바이트 기반 출력  OutoutSteam;
1)InputStream
상속을 받아서 다양한 스트림을 제공하고 있으며 개발환경과 상황에 따라 목적을 갖고 다양한 메소드를 오버라이딩 해서 계층 구조를 사용하게 된다.
fileInputStream
파일에서 바이트 단위로 자료를 읽어 들일 때 사용하는 스트림이다.
이미지, 동영상, 텍스트 모든 타입의 파일들을 읽어올 수 있다.
1)파일을 반복ㄷ하는 횟수를 줄일 때는 버퍼의 길이만큼 전체의 값을 늘려줄 수 있다.
 read():1byte -->read(byte[] b)

FilterArrayInputStream



2)OutputStream
바이트 기반의 최상위 클래스 OutputStream 객체로 제공되며, 상속 다양한 출력 스트림이 존재한다.
-FileOutputStream
파일을 버퍼단위로 읽어 들이면서 여러가지 파일들의 내용을 출력하는 방법을 사용한다.
new FileOutputStream(경로/파일명,이어쓰기옵션):일반적인 파일 만드는 방법.
이어쓰기 옵션 : true  : 이어쓰기./ false : 새로만든다.
이미지를 복사해서 패키지안에서 새롭게 이미지를 생성하는법.
read(byte[]) :  단위로 묶어야 큰 파일은 성능이 빨라진다.



2. 문자 기반 스트림.
문자를 기반으로 하기 때문에 텍스트 문서만을 읽고 쓸 수 있당.
문자기반 입력 스트림의 최상위 클래스 Reader가 있으며, 문자 기반 출력스트림의 최상위 클래스는 writer이다.

1)Reader  : 문자 입력 스트림.
-BufferedReader
-CharArrayReader
-InputStreamReader>FileReader

2)Writer : 문자 출력 스트림.
문자기반의 출력 스트림이고 최상위 클래스는 Writer 클래스를 상속해 다양한 클 래스를 제공함.
-BufferedWrite;
-CharArrayWrite
-InputStreamWriter : FileWriter


3. 보조 스트림.
보조 스트림은 실제로 데이터를 주고 받지 않고 Stream의 기능을 향상 시키거나 새로운 기능을 제공해주는 Stream으로 다른 보조 스트림과 중첩하여 사용한당.
-기반 스트림 : 대상에 직접적으로 자료를 읽고 쓰는 스트림;
-보조 스트림 : 직접 읽고 쓰는 기능은 없지만 기반 스트림에 추가로 사용하는 스트림이당.

1)성능 향상 보조 스트림.
하드 디스크나 네트워크 입출력은 성능에 영향을 주기 때문에 보조 스트림을 이용해 버퍼에 메모리 작업을 시켜서 성능을 향상시킬 수 있다.
-바이트기반

BufferedInputStream/BufferedOutputStream(바이트 기반)
프로그램 장치로부터 데이터를 직 접 읽는 대신 버퍼에 저장해 두었다가 큰 단위로 받을 수 있고, 출력에 대한 보조 스트림은 프로그램에서 전송한 데이터를 내부 버퍼에 쌓아 두었다가 꽉차면 모든 데이터를 한꺼번에 저장한다.

-문자 기반
BufferedReader/BufferedWriter(문자기반)


2)문자 변환 보조 스트림.
바이트 기반 스트림은 글자를 읽을 경우 한글일 경우 정상적으로 출력되지 않는다.
-InputStreamReader
바이트 기반 스트림. InputStream을 문자 기반 스트림ㄴ Reader로 변환하는 보조스트림이당.
(바이트-보조스트림-문자스트림)
-OutputStreamReader
바이트 기반 OutputStream을 문자 기반 Writer로 변환하는 보조 스트림이다.
바이트-보조스트림-문자 기반 스트림.

@ File class
파일 및 폴더에 대한 정보들을 파악하는 메서드이다.
물리적인 파일이나 폴더를 다양한 메 서드로 속성에 대한 정보 값을 지원하기 위해서 사용된다.

@직렬화/역직렬화.
자바 시스템 내부에 사용하는 객체나 데이터를 외부의 시스템에서 사용할 수 있도록 바이트 단위의 데이터로 변환하는 기술로 바이트로 변환된 데이터를 다시 객체로 변환하는 기술이다.
-SerialVersionUID : 인터페이스로 상속받아서 코드값을 만든다.
기반 스트림 : FileOutputStream/FileInputStream
직렬화 : 객체를 데이터로 변환하는 방법 (ObjectOutputStream);
역직렬화 : 데이터를 객체로 벼 ㄴ환하는 방법(ObjectInputStream);
