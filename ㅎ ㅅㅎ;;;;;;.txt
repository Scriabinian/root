설계도를 구현하고 설계도 안에 대표할 수 있는 설명서를 기술(기능=속성)
-클래스 정의 : 클래스란 객체를 정의한것.
-클래스의 용도 : 클래스는 객체를 생성하는데 사용됨.
-객체의 정의 : 실제로 존재하는것. 사물 또는 개념
-객체의 용도 : 객체가 가지고 있는 기능과 속성에 따라 다르당.
유형의 객체 - 책상, 의자, 자동차, TV, 스마트폰.

*클래스(설계도)만 잘 만들어 놓는다면 복잡한 제품이라도 객체를 생성해서 사용하면 된당.

2. 객체의 구성요소(속성과기능)
객체는 다른 속성과 기능들의 집합임(생성할때마다 다름.)
속성 : 변수
기능 : 함수.

@클래스의 구성.
클래스 설계도에 들어갈 수 있는 료소이다.
생성자로 접근해서 클래스 내부에 필드(변수)와 메소드(함수)를 정의하고 사용했당.
1)필드 : 객체의 데이터 상태를 저장하는 변수.
-필드의 자료형은 기본자료형과 같당.( int = 0);
-필드를 접근하는 방법은 객체 생성해서 접근함 ^^.

객체명 : 클래스르 이용해 만든 객체이름.
필드명 : 만든 객체가 가지고있는 필드이름.

2)메서드 : 객체의 기능을 담당함.
메서드 호출하면 기능수행할 수 있고 리턴값 있을수도있다.
따라서 결과값 출력가능
반환타입 메소드명(인수){

}
3) 생성자 : 클래스를 호출해서 접근할 수 있는 대상.
@정적or스태틱 멤버/인스턴스 멤버/지역변수 ㅎ ㅅㅎ.
1)인스턴스 (멤버변수:IV)
-모든 객체가 같은 필드값을 가질 수 있슴.
-인스턴스를 생성하면 멤버변수의 값을 변화시켜서 따로 선언 가능.
-독립적인 저장공간을 가지게 되며 고유한 속성 유지.

2)정적 (클래스) 멤버 변수 (CV)
-객체가 아닌 클래스의 의존적
-객체를 생성하지 않고 변수 사용.
-객체르 생성할때 메모리에 올라가는 것이 아니라 프로그램 시작할 떄 메모리가 적재된다.
-전역변수의 성격을 갖고 모든 인스턴스가 공통된 저장공간을 갖게됨.

3)지역변수
-메서드 영역이고, 메서드 내에서 선언된 블록에서만 사용가능. 블록을 벗어나면 소멸.

@메서드
1)메서드의 정의
특정 기능을 수행하기 위해 코드를 하나의 블록으로 묶어 놓는것이다.
1)문장들을 블록으로 묶어놓은것.
2)값을 받아서 처리하고, 결과를 반환.
3)메서드의 장점
-중복감소
-관리쉬움
-재사용성
-간결함.
4)작성지침
-반복적으로 수행되는 문장을 작성.
-하나의 메서드는 한가지 기능만 수행하도록
-int add(int x,int y)
-int add(int x,y){}->매개변수 선언 잘못.

3)메서드의 구조(정적메소드, 멤버메소드)
선언부 : header
-매개변수 : 입력되는 public int add(int a, double b)
-반환타입 : 리턴값과 반환타입 일치해야함.
return int 형 === header 반환타입 동일하게 맞춘다.
구현부 : body
-지역변수(IV):메서드 내에 선언된 변수.
